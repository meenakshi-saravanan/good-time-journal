# Bug Fix Specification – Homepage Entry Selection Refactor

## Objective

Refactor the homepage entry list implementation before implementing the preview panel.

The current homepage successfully renders journals and entries, but the JavaScript architecture has accumulated duplicate rendering functions and incomplete selection logic.

The goal of this task is to cleanly establish the homepage interaction architecture.

---

# Scope

This task only affects the homepage.

Do **not** modify:

* Backend
* Database
* API routes
* Authentication
* Journal Entry page
* Tiptap editor

Only modify:

* `public/js/app.js`
* `public/js/journal.js`
* `public/css/style.css`

---

# Current Problems

## 1. Duplicate renderEntryList()

`journal.js` currently contains **two** implementations of:

```javascript
renderEntryList(entries)
```

The first implementation renders:

* `.entry-card`
* `.entry-card-header`
* `.entry-card-title`

The second implementation renders:

* `.entry-list-item`
* `.entry-card-top`
* `.entry-list-title`

Because JavaScript keeps only the last declaration, the first implementation is dead code.

### Required Fix

* Remove the obsolete implementation.
* Keep only one `renderEntryList()` function.
* Ensure there is a single rendering path for homepage entries.

---

## 2. Entry Selection Does Not Exist

Currently clicking an entry either:

* navigates to another page, or
* performs no action.

The homepage should maintain selection state.

### Expected Behaviour

Clicking an entry should:

1. Store

```javascript
appState.selectedEntryId
```

2. Re-render the entry list.

3. Highlight the selected entry.

4. Trigger preview rendering.

No page navigation should occur.

---

## 3. Journal Switching Does Not Refresh Entry List

Current implementation:

```javascript
selectJournal()

↓

selectedJournalId updated

↓

renderJournalSidebar()
```

The middle panel is not consistently refreshed.

### Required Behaviour

After changing journals:

```text
Journal Click

↓

selectedJournalId updated

↓

getFilteredEntries()

↓

renderEntryList()
```

No additional API request should be made.

Filtering should use the entries already stored in memory.

---

## 4. Missing Selected State

The rendered entry cards have no selected appearance.

### Required Behaviour

When

```javascript
entry.id === appState.selectedEntryId
```

the entry card should receive

```css
.selected
```

and display a visual selected state.

---

## 5. Preview Pipeline Preparation

Do **not** implement the preview UI.

Only prepare the architecture.

After an entry is selected:

```text
Entry Click

↓

selectEntry()

↓

renderEntryPreview(entry)
```

The preview function may remain a placeholder.

The goal is only to establish the interaction flow.

---

# Required Architecture

The homepage should follow this pipeline.

```text
Homepage

↓

loadJournals()

↓

loadEntries()

↓

appState

    journals

    entries

    selectedJournalId

    selectedEntryId

↓

renderJournalSidebar()

↓

renderEntryList()

↓

selectEntry()

↓

renderEntryPreview()
```

There should be one clear rendering path.

---

## public/js/journal.js

### Remove

* Duplicate `renderEntryList()`

### Update

Each entry card should:

* register a click handler

