# Bug Report: Flash of Good Time Journal Fields on Entry Creation Page

## Issue

When a user clicks **"Add New Entry"**, the page initially displays the **Good Time Journal** fields for a brief moment before hiding them and showing the correct journal entry form.

This creates a noticeable UI flicker and makes the application feel slow or unstable.

---

## Expected Behavior

When the page loads:

* The user should only see the entry form associated with the selected journal.
* No incorrect fields should appear, even for a fraction of a second.
* The page should render in its final state immediately after initialization.

---

## Current Behavior

Page load sequence:

```text
User clicks "Add New Entry"
↓
Page loads
↓
Good Time Journal fields appear briefly
↓
Journal type is resolved in JavaScript
↓
Good Time Journal fields are hidden
↓
Correct fields are displayed
```

This results in a flash of incorrect content (FOIC).

---

## Root Cause

The HTML is rendering before the journal type has been determined.

At initial render:

```text
goodTimeEntryFields = visible
standardEntryFields = hidden
```

The browser paints the page immediately.

After that:

```javascript
loadJournal();
determineJournalType();
hide/show appropriate fields;
```

runs and updates the UI.

Because the browser has already rendered the page, the user sees the wrong content briefly.

---

## Recommended Fix

### Option 1 (Preferred)

Hide all journal field sections by default.

Initial HTML:

```html
<div
  id="goodTimeEntryFields"
  class="d-none"
>
</div>

<div
  id="standardEntryFields"
  class="d-none"
>
</div>
```

After journal data is loaded:

```javascript
if (journal.template === "good-time") {
  goodTimeEntryFields.classList.remove("d-none");
} else {
  standardEntryFields.classList.remove("d-none");
}
```

This prevents any incorrect content from rendering.

---

## Option 2 (More Robust)

Hide the entire form until initialization completes.

Initial HTML:

```html
<form
  id="entryForm"
  class="d-none"
>
</form>
```

After:

```javascript
await loadJournal();
await initializeEntryForm();
```

show the form:

```javascript
document
  .getElementById("entryForm")
  .classList.remove("d-none");
```

---

## Option 3 (Best UX)

Display a lightweight loading state while the journal type is being resolved.

Example:

```html
<div id="entryLoading">
  Loading journal...
</div>

<form
  id="entryForm"
  class="d-none"
>
</form>
```

After initialization:

```javascript
document
  .getElementById("entryLoading")
  ?.remove();

document
  .getElementById("entryForm")
  .classList.remove("d-none");
```

---

## Acceptance Criteria

* No Good Time Journal fields appear when opening a standard journal.
* No field flickering occurs during page load.
* Entry form remains hidden until journal type is known.
* Correct journal fields are displayed on first render.
* Works consistently for all current and future journal templates.
* No regression to existing entry creation functionality.
